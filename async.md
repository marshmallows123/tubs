## 1. Фундаментальные принципы

### JavaScript — однопоточный
Код выполняется в одном потоке. Долгие операции (сетевые запросы, таймеры, работа с файлами) не могут блокировать интерфейс или другие задачи.

### Событийный цикл (Event Loop)
Браузер / Node.js предоставляют движку JS «хостинг-среду» с дополнительными возможностями. Работает это так:

1. **Стек вызовов (Call Stack)** — место, где выполняются синхронные функции. Последняя вошла — первая вышла.
2. **Web API / Libuv** — таймеры, AJAX, файловые операции и т.п. Передают коллбэки в очередь задач.
3. **Очередь макрозадач (Task Queue / Macrotask Queue)** — `setTimeout`, `setInterval`, I/O, события, UI-рендеринг. Эти коллбэки ждут, когда стек опустеет.
4. **Очередь микрозадач (Microtask Queue)** — `Promise.then/catch/finally`, `queueMicrotask`, `MutationObserver`. Имеют более высокий приоритет, выполняются сразу после каждого освобождения стека, **до** следующей макрозадачи.

**Алгоритм одной итерации цикла:**
- Взять одну макрозадачу (например, коллбэк `setTimeout`) и выполнить её;
- Выполнить **все** микрозадачи из очереди, включая те, что добавились во время этого выполнения;
- Обновить экран (если браузер);
- Перейти к следующей макрозадаче.

## 2. Основные инструменты асинхронности

- **Коллбэки** — передача функции в асинхронный вызов (`setTimeout`, `addEventListener`). Минусы: «callback hell».
- **Promise** — объект-обёртка будущего значения. Позволяет выстраивать цепочки `.then().catch()`. Создаёт микротаски.
- **async/await** — синтаксический сахар над `Promise`. `await` приостанавливает выполнение функции, не блокируя поток, и «ждёт» разрешения промиса. Код после `await` оборачивается в микрозадачу.
- **queueMicrotask** — явно добавляет задачу в очередь микрозадач.
- **setTimeout(cb, 0)** — откладывает `cb` в очередь макрозадач (даже с нулевой задержкой).

## 3. Интерактивная инструкция: разбор порядка выполнения

Скопируйте этот код в среду Node.js (или консоль браузера). Внимательно читайте комментарии – каждая строчка объяснена. В конце вы увидите полный вывод и его детальный разбор.

```javascript
// ======== ДЕМОНСТРАЦИЯ АСИНХРОННОСТИ ========
console.log('1. Синхронный старт');

// ---- Макрозадача: setTimeout ----
setTimeout(() => {
  console.log('2. Макрозадача setTimeout(0)');
}, 0);

// ---- Микрозадача: Promise.then ----
Promise.resolve()
  .then(() => {
    console.log('3. Микрозадача Promise.then #1');
  });

// ---- Микрозадача: вложенные микрозадачи (Promise внутри .then) ----
Promise.resolve()
  .then(() => {
    console.log('4. Микрозадача #2 (внешняя)');
    // Внутри микрозадачи создаём ещё одну микрозадачу
    Promise.resolve().then(() => {
      console.log('5. Вложенная микрозадача внутри #2');
    });
  });

// ---- Асинхронная функция с await ----
async function demoAsync() {
  console.log('6. Начало async-функции (синхронная часть)');

  // await превращает всё последующее в микрозадачу, аналогичную .then
  await Promise.resolve('значение');

  console.log('7. После await (микрозадача)');
}
demoAsync();

// ---- Макрозадача: setImmediate (только в Node.js) ----
// В браузере можно заменить на setTimeout, но для наглядности оставлю:
if (typeof setImmediate === 'function') {
  setImmediate(() => {
    console.log('8. Макрозадача setImmediate (перед следующим setTimeout)');
  });
} else {
  setTimeout(() => {
    console.log('8. Макрозадача setTimeout(0) вместо setImmediate');
  }, 0);
}

// ---- Ещё одна синхронная строка ----
console.log('9. Синхронный финиш до обработки очередей');

// ---- Создание цепочки из нескольких микрозадач ----
Promise.resolve()
  .then(() => {
    console.log('10. Микрозадача #3');
    return 'значение';
  })
  .then((val) => {
    console.log('11. Микрозадача #4, получено:', val);
  });

// ---- Дополнительная макрозадача с нулевой задержкой ----
setTimeout(() => {
  console.log('12. Вторая макрозадача setTimeout(0)');
}, 0);

// ---- Явная микрозадача через queueMicrotask ----
queueMicrotask(() => {
  console.log('13. Явная микрозадача (queueMicrotask)');
});

// ---- Глубокая вложенность микрозадач ----
Promise.resolve()
  .then(() => {
    console.log('14. Микрозадача #5');
    queueMicrotask(() => {
      console.log('15. Микрозадача внутри queueMicrotask из #5');
    });
  });

console.log('16. Синхронный конец скрипта');
```

## 4. Фактический вывод (Node.js / современный браузер)

**Порядок сообщений в консоли:**
```
1. Синхронный старт
6. Начало async-функции (синхронная часть)
9. Синхронный финиш до обработки очередей
16. Синхронный конец скрипта
3. Микрозадача Promise.then #1
4. Микрозадача #2 (внешняя)
5. Вложенная микрозадача внутри #2
7. После await (микрозадача)
10. Микрозадача #3
11. Микрозадача #4, получено: значение
13. Явная микрозадача (queueMicrotask)
14. Микрозадача #5
15. Микрозадача внутри queueMicrotask из #5
2. Макрозадача setTimeout(0)
8. Макрозадача setTimeout(0) вместо setImmediate   (или 8. setImmediate)
12. Вторая макрозадача setTimeout(0)
```

> **Важно:** В Node.js `setImmediate` выполнится до второго `setTimeout`, но после первого. В браузере без `setImmediate` все `setTimeout` идут в порядке добавления, если не было задержек.

## 5. Пошаговый разбор произошедшего

### Синхронная фаза (стек)
- Выводятся `1`, `6`, `9`, `16`.
- В это время регистрируются все асинхронные вызовы, но их коллбэки попадают в очереди:  
  **Макрозадачи:** два `setTimeout(fn,0)` + `setImmediate`(в Node).  
  **Микрозадачи:** цепочки промисов, `await`, `queueMicrotask`.

### Микротаски первой волны (после опустошения стека)
Движок запускает **все** накопившиеся микрозадачи:
1. `Promise.then #1` → `3`.
2. `Микрозадача #2 (внешняя)` → `4`, а внутри неё добавляется `Promise.then()` → в очередь микрозадач.
3. Очередь сейчас не пуста, поэтому сразу выполняется вложенная → `5`.
4. Код после `await` в `demoAsync()` → `7`.
5. `Микрозадача #3` → `10`, возвращает значение; её `.then` попадает в очередь и сразу выполняется → `11`.
6. `queueMicrotask` → `13`.
7. `Микрозадача #5` → `14`, внутри вызывает `queueMicrotask` → `15` сразу же исполняется.

Теперь очередь микрозадач пуста.

### Макрозадачи
Цикл берёт **одну** макрозадачу:
- Первый `setTimeout(0)` → `2`.
- После этого снова проверяются микрозадачи (пусто).
- Затем либо `setImmediate` (Node) → `8`, либо второй `setTimeout` (браузер) → `8`.
- Затем оставшийся `setTimeout` → `12`.

## 6. Ключевые правила, которые стоит запомнить

1. **Сначала весь синхронный код** до конца текущего «тикера» (скрипта / модуля).
2. **Все микрозадачи** выполняются до следующей макрозадачи. В том числе те, что порождены внутри других микрозадач.
3. **Макрозадачи** выполняются по одной за цикл, с проверкой микротаск после каждой.
4. `async/await` ведёт себя так же, как `Promise.then`: `await` превращает код ниже в микрозадачу, аналогичную `.then()`.
5. `queueMicrotask(fn)` ≈ `Promise.resolve().then(fn)`.
6. Даже `setTimeout(…, 0)` не гарантирует мгновенного выполнения — он будет ждать завершения всех текущих микрозадач.
